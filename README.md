## Что за проект?

**Hub** — это платформа для создания документации, базы знаний, гайдов, changelog-порталов и продуктовых лендингов на базе Markdown.

Ключевая идея: вы работаете с привычным контентом (`.md`/`.mdx`-подобный опыт без сложного MDX), а Hub автоматически превращает его в современный сайт:

- с удобной структурой и автогенерируемой навигацией,
- с интерактивными таблицами и блоками,
- с поддержкой графиков и формул,
- с встроенным CMS-like редактором и live preview,
- с подготовкой к SEO/GEO и мультиязычным сценариям.

Hub подходит как для личных проектов, так и для небольших команд, которым нужен быстрый цикл «написал → опубликовал → обновил» без тяжёлой enterprise-инфраструктуры.

---

## Ключевые особенности: почему стоит выбрать Hub

### 1) Высокая скорость запуска

- Не нужно поднимать отдельный сложный CMS-бэкенд.
- Контент — в Markdown, структура — в файловой системе.
- Быстрое масштабирование от одной страницы до большого docs-портала.

### 2) Контент «богаче», чем обычный Markdown

- Продвинутые таблицы с фильтрацией и поиском.
- Блоки уведомлений (note/tip/warning и др.).
- Графики прямо из Markdown-данных.
- KaTeX-формулы для технической документации.

### 3) Отличный UX для читателя

- Автосайдбар, breadcrumbs, TOC на странице.
- Прогресс чтения, lightbox для изображений.
- Тёмная/светлая темы и mobile-first навигация.

### 4) Отличный UX для автора

- Встроенная CMS-like панель.
- Live preview при редактировании.
- UI playground для демонстрации и настройки компонентов.

### 5) Практичный фокус продукта

Hub закрывает основной слой задач для контент-проектов и документации без перегруза сложными enterprise-модулями (SSO/SAML, audit-log и т.д.).

## Для кого

Hub особенно полезен для:

- **Технических писателей** — продуктовая документация, onboarding, how-to, API-гайды.
- **Разработчиков и DevRel** — docs-центры, примеры, changelog, migration guides.
- **Продуктовых команд** — help center, feature docs, внутренние регламенты.
- **Основателей и инди-хакеров** — быстрый запуск контент-хаба без тяжёлого стека.
- **Малых команд** — когда важны скорость публикации и простота поддержки.

> Если вам нужна тяжёлая корпоративная модель (глубокие роли, SSO/SAML, audit trail, enterprise SLA), Hub сейчас не про это.

## Killer Features

### 🔥 Таблицы на стероидах

Стандартная Markdown-таблица превращается в интерактивный компонент:

- **Мультиселект-фильтры** по любому столбцу
- **Поиск с подсветкой** совпадений
- **Скрытие колонок** на лету
- **Fullscreen-режим** для широких таблиц
- **Drag-to-scroll** для горизонтальной прокрутки
- **Копирование** в Markdown или Excel одним кликом
- **HTML в ячейках** — бейджи, ссылки, иконки

Всё это из обычного `| col | col |` синтаксиса.

### 🔥 Markdown Blocks — богатый контент без MDX

[Поддерживаемые блоки](https://opensophy.com/docs/markdown-guide/)

### 🔥 Live UI Component Playground

Каждый UI-компонент платформы доступен в интерактивном playground прямо в документации. Меняйте `scale`, `rotation`, `blur`, `gradient`, скорость анимации — и видьте результат в реальном времени.

### 🔥 CMS-like панель редактирования

Встроенный редактор с live preview — правите Markdown и сразу видите результат. Создание страниц, секций и категорий прямо из UI. Никакого деплоя ради одной правки.
