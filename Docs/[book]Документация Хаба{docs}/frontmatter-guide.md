# Обязательные поля

## `title`

**Тип:** `string`

Главный заголовок статьи. Используется в:

- шапке документа
- результатах поиска
- метатегах страницы
- оглавлении сайдбара

```yaml
title: "Полное руководство по Frontmatter"
````

---

## `description`

**Тип:** `string`

Краткое описание содержимого документа (обычно 1–2 предложения).

Используется в:

* результатах поиска
* карточках документов
* метатеге `<meta name="description">` для SEO
* Open Graph превью при расшаривании

```yaml
description: Справочник по всем доступным полям frontmatter с примерами использования и рекомендациями.
```

---

# Необязательные поля

## `type`

**Тип:** `string`

Определяет тип контента для формирования структуры URL.

Если поле указано:

```
/{type}/{slug}
```

Если не указано:

```
/{slug}
```

Примеры значений:

* `docs` — документация
* `blog` — блог
* `news` — новости
* `guides` — руководства
* `tutorials` — туториалы

```yaml
type: docs
```

---

## `typename`

**Тип:** `string`

Человекочитаемое название категории.

Отображается в:

* сайдбаре
* бейдже над заголовком статьи
* breadcrumbs
* результатах поиска

```yaml
typename: Документация
```

---

## `author`

**Тип:** `string`

Имя автора статьи.

Отображается:

* в шапке документа
* в фильтрах поиска

```yaml
author: veilosophy
```

Несколько авторов:

```yaml
author: veilosophy, opensophy
```

---

## `date`

**Тип:** `string`
**Формат:** `YYYY-MM-DD`

Дата публикации документа.

Используется для:

* сортировки документов
* отображения в шапке статьи
* фильтрации по времени
* генерации `sitemap.xml`

```yaml
date: 2026-02-18
```

---

## `updated`

**Тип:** `string`
**Формат:** `YYYY-MM-DD`

Дата последнего обновления документа.

Если указана — отображается как:

> Обновлено: {дата}

```yaml
updated: 2026-03-15
```

---

## `tags`

**Тип:** `string` (список через запятую)

Теги для категоризации и поиска.

Используются в:

* расширенном поиске
* карточках документов
* блоке связанных статей

```yaml
tags: Markdown, форматирование, руководство, синтаксис
```

---

## `icon`

**Тип:** `string`

Название иконки из **Lucide React**.

Используется:

* рядом с заголовком в сайдбаре
* в карточках документов

Популярные иконки:

* `book` — документация
* `file-code` — техническая статья
* `lightbulb` — идеи и советы
* `shield` — безопасность
* `rocket` — новые функции
* `crown` — важное

Полный список:

```
https://lucide.dev/icons
```

```yaml
icon: file-code
```

---

## `category`

**Тип:** `string`

Дополнительная категоризация.

Используется для:

* внутренней группировки
* фильтрации
* аналитики

Не отображается напрямую в интерфейсе.

```yaml
category: hub
```

---

## `keywords`

**Тип:** `string` (список через запятую)

SEO-ключевые слова.

Добавляются в метатег:

```html
<meta name="keywords">
```

Помогают поисковым системам индексировать страницу.

```yaml
keywords: Markdown руководство, форматирование текста, синтаксис Markdown, документация Hub
```

---

## `canonical`

**Тип:** `string` (URL) или `null`

Указывает каноническую версию страницы.

Используется если контент доступен по нескольким URL.

```yaml
canonical: https://hub.opensophy.com/docs/original-article
```

Если дубликатов нет:

```yaml
canonical: null
```

---

## `robots`

**Тип:** `string`

Инструкции для поисковых роботов.

Добавляется в метатег:

```html
<meta name="robots">
```

Возможные значения:

| Значение            | Описание                                       |
| ------------------- | ---------------------------------------------- |
| `index, follow`     | индексировать страницу и переходить по ссылкам |
| `noindex, follow`   | не индексировать, но переходить по ссылкам     |
| `index, nofollow`   | индексировать, но не переходить по ссылкам     |
| `noindex, nofollow` | не индексировать и не переходить               |

```yaml
robots: index, follow
```

---

## `lang`

**Тип:** `string` (ISO 639-1)

Язык документа.

Используется для:

* HTML-атрибута `<html lang="">`
* корректной индексации
* accessibility и скринридеров

```yaml
lang: ru
```

Другие примеры:

```yaml
lang: en
lang: de
lang: fr
```

# Иконки и NavPopover

Пример структуры:

```
Docs/[book]Документация{docs}/article.md
```
где:

[book] - иконка
Документация - это название категории или nav popover
{docs} - slug который будет в ссылке.(написав это вы создатите nav popover) 

Если просто нужна иконка в категории: [book]Документация

# Рекомендации по использованию

## Минимальный frontmatter

Для простого документа достаточно:

```yaml
---
title: "Название статьи"
description: Краткое описание содержимого статьи.
---
```

---

## Стандартный frontmatter для документации

```yaml
---
title: "Название статьи"
description: Краткое описание содержимого статьи.
type: docs
typename: Документация
author: veilosophy
date: 2026-02-18
tags: тег1, тег2, тег3
icon: book
robots: index, follow
lang: ru
---
```

---

## Frontmatter для главной страницы

Пример для `welcome.md`:

```yaml
---
title: "Добро пожаловать в Hub"
description: "Hub проекта Opensophy — центр знаний"
author: ""
icon: crown
date: 2026-02-17
tags: welcome, hub, opensophy
keywords: Opensophy, Hub, документация, знания
canonical: null
robots: index, follow
lang: ru
---
```

# Важные замечания

* **Формат даты**
  Используйте `YYYY-MM-DD` (ISO 8601)

* **Теги**
  Разделяются запятыми

* **Пустые значения**

```yaml
author: ""
canonical: null
```

* **Кавычки**
  Используйте двойные кавычки для значений со специальными символами

* **Slug**
  Генерируется автоматически из имени файла и структуры папок
