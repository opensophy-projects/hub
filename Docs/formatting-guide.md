## Frontmatter — метаданные статьи

Каждая статья в Hub **обязательно** должна начинаться с блока метаданных в формате YAML, заключенного в тройные дефисы:

```yaml
---
title: "Название статьи"
description: Краткое описание статьи для SEO и превью
type: docs # или blog, news
category: Категория
author: Ваше имя
date: 2025-01-17
tags: тег1, тег2, тег3
keywords: ключевые, слова, для, SEO
bannercolor: #3b82f6
bannertext: Текст на баннере
robots: index, follow
lang: ru
---
```

### Обязательные поля

| Поле | Описание | Пример |
|------|----------|--------|
| `title` | Название статьи (отображается как H1) | `"Деплой приложений с Dokploy"` |
| `description` | Краткое описание для SEO и карточек | `"Полное руководство по установке и настройке Dokploy"` |
| `type` | Тип контента | `docs`, `blog`, `news` |
| `category` | Основная категория | `DevOps`, `Security`, `Career` |
| `author` | Имя автора | `Opensophy Team` |
| `date` | Дата публикации | `2025-01-17` |

### Опциональные поля

| Поле | Описание | Пример |
|------|----------|--------|
| `tags` | Теги через запятую | `docker, deployment, vps` |
| `keywords` | SEO ключевые слова | `dokploy, паас, деплой` |
| `bannercolor` | Цвет баннера в HEX | `#3b82f6` |
| `bannertext` | Текст на баннере | `DevOps Guide` |
| `robots` | Инструкции для поисковиков | `index, follow` |
| `lang` | Язык статьи | `ru`, `en` |

## Заголовки

Hub поддерживает 6 уровней заголовков. **Важно:** H1 (`#`) автоматически генерируется из `title` в frontmatter, поэтому в теле статьи используйте заголовки начиная с H2.

### Синтаксис

```markdown
## Заголовок 2 уровня (H2)
### Заголовок 3 уровня (H3)
#### Заголовок 4 уровня (H4)
##### Заголовок 5 уровня (H5)
###### Заголовок 6 уровня (H6)
```

### Результат

## Заголовок 2 уровня (H2)
### Заголовок 3 уровня (H3)
#### Заголовок 4 уровня (H4)
##### Заголовок 5 уровня (H5)
###### Заголовок 6 уровня (H6)

### Автоматическое оглавление

Hub автоматически генерирует оглавление (Table of Contents) на основе заголовков H2 и H3 в статье. Заголовки появляются в боковой панели для быстрой навигации.

**Рекомендации:**
- Используйте логическую иерархию заголовков
- Не пропускайте уровни (после H2 не ставьте сразу H4)
- Делайте заголовки информативными и краткими
- Используйте H2 для основных разделов, H3 для подразделов

## Форматирование текста

### Курсив (Italic)

```markdown
Текст с *курсивом* или _курсивом_
```

Текст с *курсивом* или _курсивом_

### Жирный (Bold)

```markdown
Текст с **жирным шрифтом** или __жирным шрифтом__
```

Текст с **жирным шрифтом** или __жирным шрифтом__

### Жирный курсив

```markdown
***Жирный и курсивный текст*** или ___жирный и курсивный___
```

***Жирный и курсивный текст*** или ___жирный и курсивный___

### Зачеркнутый (Strikethrough)

```markdown
~~Зачеркнутый текст~~
```

~~Зачеркнутый текст~~

### Подчеркнутый (через HTML)

```markdown
<u>Подчеркнутый текст</u>
```

<u>Подчеркнутый текст</u>

### Комбинации

```markdown
**Жирный текст с _курсивом_ внутри**
*Курсив с **жирным** внутри*
~~Зачеркнутый **жирный** текст~~
```

**Жирный текст с _курсивом_ внутри**
*Курсив с **жирным** внутри*
~~Зачеркнутый **жирный** текст~~

## Списки

### Маркированный список (Unordered List)

```markdown
- Первый пункт
- Второй пункт
- Третий пункт
  - Вложенный пункт 1
  - Вложенный пункт 2
    - Вложенный пункт 3 уровня
- Четвертый пункт
```

- Первый пункт
- Второй пункт
- Третий пункт
  - Вложенный пункт 1
  - Вложенный пункт 2
    - Вложенный пункт 3 уровня
- Четвертый пункт

Можно также использовать `*` или `+`:

```markdown
* Пункт с звездочкой
+ Пункт с плюсом
- Пункт с дефисом
```

### Нумерованный список (Ordered List)

```markdown
1. Первый пункт
2. Второй пункт
3. Третий пункт
   1. Вложенный нумерованный
   2. Еще один вложенный
4. Четвертый пункт
```

1. Первый пункт
2. Второй пункт
3. Третий пункт
   1. Вложенный нумерованный
   2. Еще один вложенный
4. Четвертый пункт

**Важно:** Для вложенных списков используйте отступ в 3 пробела или 1 таб.

### Смешанные списки

```markdown
1. Основной пункт
   - Маркированный подпункт
   - Еще один подпункт
2. Второй основной
   1. Нумерованный подпункт
   2. Еще нумерованный
```

1. Основной пункт
   - Маркированный подпункт
   - Еще один подпункт
2. Второй основной
   1. Нумерованный подпункт
   2. Еще нумерованный

## Ссылки

### Inline-ссылки

```markdown
[Текст ссылки](https://example.com)
[Ссылка с подсказкой](https://example.com "Текст подсказки при наведении")
```

[Текст ссылки](https://example.com)
[Ссылка с подсказкой](https://example.com "Текст подсказки при наведении")

### Reference-style ссылки

```markdown
Текст со [ссылкой][1] и еще одной [ссылкой][link-name].

[1]: https://example.com
[link-name]: https://another.com "Опциональная подсказка"
```

### Автоматические ссылки

```markdown
<https://example.com>
<email@example.com>
```

<https://example.com>
<email@example.com>

### Якорные ссылки (anchor links)

```markdown
[Перейти к разделу Код](#код)
```

[Перейти к разделу Код](#код)

**Правила для якорей:**
- Все буквы в нижнем регистре
- Пробелы заменяются дефисами
- Специальные символы удаляются
- Для кириллицы используется транслитерация

## Лучшие практики

### Структура статьи

**Рекомендуемая структура:**

1. **Frontmatter** с полными метаданными
2. **Введение** — краткое описание темы
3. **Основной контент** — разделен на логические секции с H2/H3
4. **Практические примеры** — код, команды, скриншоты
5. **Заключение** — итоги, ссылки на дополнительные ресурсы

### Форматирование кода

- ✅ Всегда указывайте язык для блоков кода
- ✅ Используйте информативные комментарии
- ✅ Добавляйте вывод команд, где это уместно
- ✅ Указывайте имена файлов в комментариях

```bash
# Установка зависимостей
npm install

# Запуск в dev-режиме
npm run dev

# Результат:
# > dev
# > astro dev
# 🚀 astro v4.0.0 started in 245ms
```

### Изображения и скриншоты

- ✅ Используйте осмысленные альтернативные тексты
- ✅ Оптимизируйте размер изображений (до 1MB)
- ✅ Используйте форматы WebP или PNG
- ✅ Добавляйте подписи к важным изображениям

### Таблицы

- ✅ Используйте таблицы для сравнений и структурированных данных
- ✅ Не перегружайте таблицы — максимум 5-6 столбцов
- ✅ Используйте выравнивание для улучшения читаемости
- ✅ Добавляйте заголовки для всех столбцов

### Ссылки

- ✅ Используйте описательный текст ссылок (не "нажмите здесь")
- ✅ Проверяйте работоспособность внешних ссылок
- ✅ Используйте относительные пути для внутренних ссылок
- ❌ Избегайте голых URL в тексте

**Хорошо:**
```markdown
Подробнее читайте в [официальной документации Docker](https://docs.docker.com).
```

**Плохо:**
```markdown
Читайте здесь: https://docs.docker.com
```

## Введение

Docker Compose — это инструмент для определения и запуска многоконтейнерных Docker приложений. В этом руководстве мы рассмотрим все аспекты работы с Docker Compose.

## Установка

### Linux

```bash
# Скачивание последней версии
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

# Установка прав на выполнение
sudo chmod +x /usr/local/bin/docker-compose

# Проверка установки
docker-compose --version
```

## Базовая конфигурация

Создайте файл `docker-compose.yml`:

```yaml
version: '3.8'

services:
  web:
    image: nginx:latest
    ports:
      - "80:80"
    volumes:
      - ./html:/usr/share/nginx/html
    
  db:
    image: postgres:15
    environment:
      POSTGRES_PASSWORD: secret
    volumes:
      - pgdata:/var/lib/postgresql/data

volumes:
  pgdata:
```

## Основные команды

| Команда | Описание |
|---------|----------|
| `docker-compose up` | Запуск всех сервисов |
| `docker-compose up -d` | Запуск в фоновом режиме |
| `docker-compose down` | Остановка и удаление контейнеров |
| `docker-compose ps` | Список запущенных сервисов |
| `docker-compose logs` | Просмотр логов |
