# Contributing to Opensophy Hub

## Setup

```bash
npm install
npm run dev
```

## Before opening a PR

Run checks locally:

```bash
npm run lint
npm run build
npm run check
```

## PR rules

1. Keep PRs focused and small.
2. Add/update docs for user-facing changes.
3. Include test/check commands in PR description.
4. Use clear commit messages (e.g. `docs: ...`, `feat: ...`, `fix: ...`).
